# 멍구와의 약속 예약하기

## 주요 기능
1. 멍구는 가능한 약속 시간을 등록할 수 있습니다.
2. 사용자는 멍구의 약속시간에 대하여 예약할 수 있습니다.
    - 에약이 생기면 관리자인 멍구에게 예약 정보 이메일이 발송됩니다.
3. 사용자는 예약 기록을 확인할 수 있습니다.
4. 사용자는 특정 예약 상세 정보를 확인할 수 있습니다.

## 도메인 용어

PlanDate
- 약속 날짜

TimeSlot
- 가능한 시간 슬롯
- 약속의 시작 시간 - 끝나는 시간을 의미

ReservationSlot
- 예약 슬롯
- 약속 날짜와 시간 슬롯으로 하나의 예약 슬롯을 나타낼 수 있음

- Reservation
- 예약
- 예약자와 예약 슬롯으로 하나의 예약을 나타낼 수 있음

## 구현한 기능

admin-reservation
- [ ] 관리자는 예약슬롯을 생성할 수 있다.
- [ ] 관리자는 예약 현황을 모두 조회할 수 있다. 

user-reservation
- [x] 사용자는 예약을 등록할 수 있다.
  - 등록 후 관리자에게 이메일을 발송한다.
- [x] 사용자는 예약 기록들을 조회할 수 있다.
- [x] 사용자는 예약 상세정보를 조회할 수 있다.

email-send
- [x] 예약 생성 시 예약 정보를 관리자에게 전송한다.
    - 현재 관리자는 멍구: wooung@duskung.ac.kr
    - PostMark API를 활용하여 구현

user
- [x] 유저는 로그인할 수 있다.
- [x] 관리자와 사용자인지 Jwt로 검사한다.
- [x] 관리자만 이용할 수 있는 API는 권한검사를 진행한다.
