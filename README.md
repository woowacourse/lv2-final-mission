# 🚇 지하철 좌석 예약 서비스

2025년 6월, **서울시**에서 지하철 좌석 예약 시스템을 일부 차량에 한해 시범 운영합니다.  
현재 시범 대상은 **2호선 1-1칸의 5개 좌석**입니다.  
따라서 현재는 **2호선의 1-1칸만 예약이 가능**합니다.

---

## ✅ 주요 기능

1. 사용자는 지하철의 특정 좌석을 예약할 수 있습니다.
2. 모든 사용자는 전체 예약 현황을 확인할 수 있습니다.
3. 사용자는 본인의 예약 현황을 상세하게 확인할 수 있습니다.
4. 사용자는 본인의 예약을 **수정/삭제**할 수 있습니다.
5. 예약 시 해당 선로에 하나라도 이미 예약한 사람이 있는지 중복 검사합니다.

---

## 🧠 도메인 지식

- 열차 번호는 `1번`부터 `9번`까지 존재한다고 가정합니다.
- 예약은 **1-1칸**만 대상으로 하며, 해당 칸에는 **5개의 좌석(A~E)** 이 있습니다.
- 각 좌석은 `A`, `B`, `C`, `D`, `E`로 구분됩니다.
- 2호선은 **순환선**이며, 사용자는 **최단 거리 방향**으로 이동한다고 가정합니다.

> ⚠️ 추후 확장을 위해 [열차 운행 시간은 오전 6시 ~ 자정(0시)] 으로 고려하고 있습니다. (현재 기능에선 제외)

---

## 📦 구성 요소

- **지하철 (Subway)**: 호선(Line)과 열차 번호를 포함합니다.
- **역 (Station)**: 호선(Line)과 역 이름(Name)으로 식별됩니다.
- **예약 (Reservation)**: 지하철 ID(Subway FK)와 좌석 정보를 포함합니다.
- **좌석 (Seat)**: 예약된 좌석의 정보를 포함합니다.
- **멤버 (Member)**: 휴대폰 번호와 이름을 포함합니다.
- **호선 (Line)**: 지하철 호선의 정보를 포함합니다.

---

## 🔐 인증 / 인가 (Authentication & Authorization)

- **JWT 기반 인증 방식** 사용
- 회원가입: 휴대폰 번호 입력 → 인증 → 랜덤 이름 생성 → 토큰 발급
- 로그인: 휴대폰 인증 후 DB 확인 → 등록된 번호일 경우 토큰 발급
- 인가: 요청 시 JWT 토큰 유효성 검사
